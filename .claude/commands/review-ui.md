아래 파일을 UI/UX 관점에서 코드 리뷰해줘: $ARGUMENTS

다음 항목을 기준으로 검토하고, 각 항목별로 **현재 상태**, **문제점**, **개선 제안**을 작성해줘:

1. **반응형 디자인** — 모바일/태블릿/데스크탑 breakpoint 대응 여부 (Tailwind `sm:`, `md:`, `lg:` 활용)
2. **접근성(a11y)** — aria 속성, 시맨틱 HTML, 키보드 탐색, 색상 대비
3. **컴포넌트 재사용성** — props 설계, 관심사 분리, shadcn/ui 활용 여부
4. **시각적 일관성** — 간격(spacing), 폰트 크기, 색상 토큰 일관성
5. **인터랙션 & 피드백** — 로딩 상태, 에러 상태, hover/focus 스타일
6. **코드 가독성** — className 정리, cn() 활용, 불필요한 중복 제거

마지막에 **우선순위별 개선 목록** (High / Medium / Low)을 표로 정리해줘.
